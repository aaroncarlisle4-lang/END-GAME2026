Image Viewer — 10 Planned Changes

Overview of all features and fixes. Review and confirm scope before we write the spec.

Bug Fixes (2)

F1

Add Note button broken

Root cause found: onAddNote captures activeFullDiscriminator which is null unless the discriminator card is already expanded. Fix: refactor AddNoteModal to accept just a discriminator ID and fetch its own data — no expansion required.

F2

Import goes to wrong folder

Root cause found: The caseGroup format uses "BucketName - Label [timestamp]" and a lazy regex to extract the bucket name. If a differential name contains - (e.g. "T-cell Lymphoma"), the regex cuts off early, creating an orphan subfolder. Fix: change separator to "||" going forward + retain backward-compatible fallback. Also needs a one-time migration to re-parent existing orphaned images.

Drag & Drop (3 surfaces)

D1

Folder order (left nav)

Drag differential folders up/down in the "Differential Folders" left panel. Images follow their folder. Persists to Convex via new viewerPreferences table (per sourceType+sourceId).

D2

Image set order (thumbnail strip)

Drag image stack thumbnails up/down within the active folder. Persists by updating sortOrder on studyImages / studyManifests records in Convex.

D3

Discriminant table column order (per-case)

Drag differential columns (the diagnoses on the X-axis) left/right in the discriminant matrix. Each case stores its own column order in viewerPreferences.

Findings Everywhere (3 changes)

E1

Findings box in ALL folders (not just title)

Currently the amber "Findings" box only appears when the primary/title folder is active. It should appear for every folder, each with its own independent saved text. Schema change: add bucketName field to viewerFindings table.

E2

Import panel always visible (even if folder is empty)

Currently the thumbnail sidebar hides entirely when a folder has no images. The import button and panel must appear regardless, so you can add the first image to any folder.

E3

Findings field in the import form

Add a "Findings" textarea to the import panel. On import, the entered text is saved as that folder's findings (same as editing the Findings box manually). So you can paste findings and images in one action.

Favourites + High Yield Integration (1 feature)

V1

Heart icon → favourites → High Yield tab

Heart icon (outline → filled) on: play card thumbnail, discriminant table header, image viewer toolbar. Toggling saves to new Convex userFavourites table with sourceType + sourceId + categoryName. Favourited YJL2B cases appear in the Highest Yield tab under their category, visually distinguished with a rose/pink badge.

Caption / Credit (1 polish fix)

C1

Larger attribution text

The image attribution/credit line at the bottom of the viewer is currently very small (8–9px). Increase to a readable size (11–12px) while keeping it in the same position.
